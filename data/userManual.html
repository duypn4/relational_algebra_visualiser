1.Leaner

Step 1:

Enter your relation algebra into an input field.
You need to follow the defined grammar for inputing queries correctly.

For Select operator, just enter as normally in relational algebra and do not need to put bracket on a result relation except those from binary operators such Union, Join, etc. This is similar to Project operator.
Eg:
σ age>23 USERS
σ name="long" σ salary=1000 USERS
π name,address (USERS ⋈mms=nns WORKER)

For any binary operators, the format is stil the same as the relational algebra in database. You need to be carefull with brackets with this kind of operator. Union, Intersection and Production does need to separate by brackets cause they are commutative and associative. In contrast, Left_Join, Right_Join and Join require bracket between each pair of them because the way they are grouped will affect their intermediate results
Eg:
STAFF ⋃ STUDENT ⋃ TEACHER
(STAFF ⋈a=b STUDENT) ⋈c=d TEACHER
(STAFF ⋂ STUDENT) ⋈c=d TEACHER

If you apply this kind of operator on intermediate results from selection or projection, bracket is again not necessary. When you try to type unecessary brackets to the input field, the system will notify a syntax error.
Eg:
σ age>23 STAFF ⋂ π name,address STUDENT

Note: all comparison symbol (>,<,=,≥,≤,≠) used in the application is as same as in Java and also the string is distinguished by double quotes " surrounding it.

Step 2:

Click on "Generate Tree" button to view an output query tree.
Click on "Highlight Tree" button to see inefficencies with a input query.
Save and open the query trees by press "File" icon on the top-left corner
of the main application window.
You can zoom in and out to view the tree graph flexibly by click on the list of items on the "Tools" menu adjacent to "File" icon.
To get information and guidline, hit "Help?" next to the "File" icon.
